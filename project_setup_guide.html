1. Project Overview

An AI-powered web tool that helps students learn from construction plan drawings. Users upload a construction drawing and the system:

  1. OCR — PaddleOCR detects all text labels
  2. Circle Detection — OpenCV finds circled callouts / reference symbols
  3. LLM Explanations — GPT-4o explains each term with construction domain expertise
  4. RAG Dictionary — Matches terms against RSMeans Illustrated Construction Dictionary (ChromaDB vector search + JSON exact lookup)
  5. VLM Analysis — GPT-4o Vision analyzes full drawings or user-selected regions
  6. Google Image Search — Fetches related construction images
  7. Activity Logging — Tracks all user interactions for research

7. New Device Setup — Step by Step

Option A: Native Setup (Development)

Prerequisites

  • Python 3.10+ — python --version
  • Node.js 18+ — node --version
  • Git — for cloning

1 Clone the repo

git clone https://github.com/anidude09/Gen_ai-for-stem-education.git
cd Gen_ai-for-stem-education

2 Create Python virtual environment

Windows (PowerShell):

python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install --upgrade pip

Windows (Git Bash):

python -m venv .venv
source .venv/Scripts/activate
pip install --upgrade pip

Mac / Linux:

python3 -m venv .venv
source .venv/bin/activate
pip install --upgrade pip

3 Install Python dependencies

pip install -r backend/requirements.txt

4 Install PaddleOCR

With NVIDIA GPU (CUDA 11.8):

pip install paddlepaddle-gpu==2.6.1.post118 -f https://www.paddlepaddle.org.cn/whl/linux/mkl/avx/stable.html
pip install "paddleocr>=2.7.0"

CPU only (Windows / Mac / no GPU):

pip install paddlepaddle==2.6.1
pip install "paddleocr>=2.7.0"
ℹ️ NoteFirst PaddleOCR import downloads ~100 MB of models. One-time cost.

5 Install the 5 local packages (order matters!)

pip install -e packages/construction_ocr
pip install -e packages/construction_circle_detector
pip install -e packages/construction_llm_explainer
pip install -e packages/construction_plan_rag
pip install -e packages/construction_vlm_analyzer

6 Set up the frontend

cd app
npm ci
npm run build
cd ..

This creates app/dist/ which the backend serves as static files.

7 Obtain the data/ folder

The data/ folder is not in the repo. Copy from an existing setup:

data/
├── Construction-Vocabulary.pdf          (~500 KB)
├── RSMeans Illustrated Construction Dictionary.pdf  (~8 MB)
└── RSMeans_Illustrated_Construction_Dictionary/
    ├── terms.json                       (~2.5 MB)
    └── images/                          (dictionary page images)
⚠️ ImportantWithout this folder, the RAG service fails. Transfer it: 
# From current machine:
tar -czf data.tar.gz data/
# On new machine (in project root):
tar -xzf data.tar.gz

8 Create .env files

Create two .env files (launcher checks both):

# Root .env
cp .env.example .env

# backend/.env — fill in your keys:
OPENAI_API_KEY=sk-your-key-here
GOOGLE_CSE_API_KEY=your-key-here
GOOGLE_CSE_CX=your-cx-here
GROQ_API_KEY=your-groq-key-here

9 Run the app

python launcher.py

Opens a Tkinter status window, starts FastAPI on http://127.0.0.1:8001, and opens the browser.

Alternative (skip the GUI):

cd backend
python -m uvicorn app:app --host 127.0.0.1 --port 8001
ℹ️ NoteFirst run takes 3-10 min as ChromaDB builds the vector store (~7500 chunks). Cached in backend/chroma_db/ after that.

Option B: Docker Setup (Production / Easier)

Prerequisites

  • Docker Desktop with WSL2 backend
  • NVIDIA Container Toolkit (optional, for GPU)
# 1. Clone
git clone https://github.com/anidude09/Gen_ai-for-stem-education.git
cd Gen_ai-for-stem-education

# 2. Place data/ folder in project root

# 3. Create .env
cp .env.example .env    # then edit with your keys

# 4. Build (~10-15 min first time)
docker compose build

# 5. Run
docker compose up

Open http://localhost:8001.

CPU-only: Remove the deploy.resources.reservations block from docker-compose.yml.

10. Common Gotchas

IssueSolution
ModuleNotFoundError: construction_ocrForgot to pip install -e packages/construction_ocr. Install all 5 local packages.
No text detected on first runPaddleOCR downloads models on first import (~100 MB). Wait for it.
ChromaDB takes foreverNormal — chunking and embedding PDFs (~7500 chunks). Takes 3-10 min.
OPENAI_API_KEY not set.env must exist in backend/ and/or project root. Check both paths.
npm ci failsDelete app/node_modules/ and try npm install instead.
OpenCV errors on MacRun brew install opencv first.
Port 8001 in useChange port in docker-compose.yml or pass --port XXXX to uvicorn.
construction_circle_detector install failsInstall construction_ocr first — it's a dependency.
